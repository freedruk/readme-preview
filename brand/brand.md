# readme-preview brand kit

## Brand idea

**Docs that don’t break.**  
Minimal, dependable, CLI-first. The product should feel like a sharp tool: small, obvious, and trustworthy.

## Voi ce

- Confident, not hypey
- Short sentences
- “Before you publish” is the key promise
- Avoid jargon unless it helps a decision

## Color

Primary:

- **Ink** `#111111`
- **Red** `#E00000`

Neutrals:

- **Slate 600** `#475569`
- **Slate 500** `#64748B`
- **Slate 200** `#E2E8F0`
- **White** `#FFFFFF`

Usage:

- Use **Ink** for almost everything.
- Use **Red** only for accents (underline, small highlights, one CTA).

## Typography

- UI Sans: `system-ui, -apple-system, Segoe UI, Roboto, Helvetica, Arial`
- Mono for commands: `ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas`

## Logo usage

- Icon is designed for 24–64px UI usage (GitHub, npm, favicons)
- Keep plenty of breathing room around it (at least 20% of icon size)

## Do / Don’t

Do:

- Lead with the 3 commands: preview / check / init
- Show 1 screenshot (or terminal snippet), not 5
- Use whitespace for hierarchy

Don’t:

- Add badges/sections above the hero on a landing page
- Use more than 2 accent colors
